Problem

Dojang is useful because it is trusted. It becomes sensitive when every dApp reads it directly.

Before

Direct address checks

A dApp checks a wallet against Dojang and may connect KYC status, balance conditions, and app behavior to the same public wallet graph.

After

Private proof checks

A dApp verifies a proof that one condition is true. The source wallet, exact balance, and personal identity stay private.

Why this matters: the more useful Dojang becomes for DeFi, RWA, quests, and verified communities, the more important selective disclosure becomes.

Solution

Make Dojang proofs private and wallet-friendly, with a path to GIWA Wallet-native UX.

Dojang proof types

Two proof types that protect what Dojang makes useful.

Verified Address

Privacy-preserving KYC proof

The current PoC proves KYC status from a trusted Dojang credential without exposing the source wallet or identity graph.

Verified Balance

Privacy-preserving threshold proof

Verified Balance follows the same privacy model next, letting dApps check thresholds without learning exact amounts or wallet history.

Proof type What dApp learns What stays private
Verified Address / KYC User is verified by an approved Dojang issuer wallet address, identity graph, raw attestation
Verified Balance User meets a threshold or eligibility rule exact balance, asset history, source wallet
Shared primitive condition result, scope, nullifier, verifier result personal identity and reusable cross-dApp identity links

Appendix A — Technical Architecture

From Dojang attestation to on-chain verification: the full proof pipeline.

Dojang / EAS Attestation source KYC attestation TX recorded on-chain via EAS
Mobile App On-device proving mopro Rust engine via UniFFI bindings
Noir Circuit ZK proof generation UltraHonk proof system (Aztec/Noir)
Verifier Contract On-chain verification Solidity verifier: verify(proof, publicInputs)
CIRCUIT

Noir ZK Circuit

  • User ownership: ECDSA signature verification
  • TX authenticity: attestation transaction parsing
  • Signer validation: Merkle proof against authorized list
  • Sybil resistance: deterministic nullifier generation
PROVING

mopro Mobile Proving

  • Rust-based prover compiled for iOS and Android
  • UltraHonk proof system with low memory mode
  • 30–60 seconds on-device proof generation
  • Circuit files downloaded and cached at runtime
VERIFY

On-chain Verifier

  • Auto-generated Solidity from Noir circuit
  • verify(bytes proof, bytes32[] publicInputs) → bool
  • View function call — no gas cost for verification
  • Deployed on Base Sepolia and Base Mainnet
Private inputs (never revealed) wallet address attestation TX user signature signer pubkey
Public inputs (shared with verifier) signal_hash signer merkle root scope nullifier
